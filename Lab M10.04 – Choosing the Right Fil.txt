Scenario 1: Streaming Clickstream Ingestión

Selected Format: JSON

Justification
	Write pattern: High-velocity streaming, JSON supports fast append writes
	Schema evolution: Flexible ideal for evolving event structures.
	Human-readable: Easy for debugging raw events.
Trade-offs: Poor compression and slower analytical queries
Alternatives: Parquet


--------------------------------------------------------------------

Scenario 2: Daily Sales Analytics

Selected Format: Parquet

Justification
	Read pattern: Analytical queries, reads only necessary columns
	Performance: Excellent for aggregations and filtering
	Storage: Highly compressed

Trade-offs: Slower writes than CSV/JSON and not human-readable

Alternatives: ORC, similar performance


-------------------------------------------------------------------


Scenario 3: Data Exchange with External Partners

Selected Format: CSV

Justification

	Interoperability: Universally supported (Excel, databases, scripts).
	Human-readable: Simple and transparent.
	Ease of use: No specialized tools required.
Trade-offs: No schema enforcement and larger storage size

Alternatives: JSON if complex data is required

------------------------------------------------------------------------

Format Selection Decision Tree:

1. What is the primary access pattern?
   - Analytical queries → Consider: Parquet, ORC
   - Transactional/Full row → Consider: CSV, JSON
   - Exploratory → Consider: CSV, JSON

2. What is the write pattern?
   - Streaming → Consider: JSON, CSV
   - Batch → Consider: Parquet, ORC
   - Append-only → Consider: Parquet, ORC

3. Is schema evolution needed?
   - Yes → Consider: JSON, Parquet
   - No → Consider: CSV, Parquet, ORC

4. What are interoperability requirements?
   - Universal → Consider: CSV, JSON
   - Specialized tools OK → Consider: Parquet, ORC

5. What is the primary concern?
   - Storage cost → Consider: Parquet, ORC
   - Query performance → Consider: Parquet, ORC
   - Human readability → Consider: CSV, JSON
   - Universal compatibility → Consider: CSV, JSON
